## 1. 模块总览

系统共划分为 **9 个业务模块** 和 **2 个公共模块**：

|编号|模块|功能概述|对应路由|
|-|-|-|-|
|M1|用户认证|手机号登录、用户注册|`/login` `/register`|
|M2|首页仪表盘|问候语、快捷入口、健康概览、每日贴士|`/home`|
|M3|健康数据管理|数据录入、列表查询、删除、CSV 批量导入|`/health-data`|
|M4|健康趋势分析|趋势图展示、统计指标、CSV 导出|`/health-trends`|
|M5|AI 健康画像|选择时间范围、AI 生成健康画像与风险评级|`/ai-profile`|
|M6|AI 健康计划|输入健康需求、AI 生成饮食/运动/睡眠建议|`/ai-plan`|
|M7|健康知识|知识库检索、AI 推送、已读标记|`/knowledge`|
|M8|健康提醒|提醒 CRUD、开关控制、全局弹窗推送|`/reminders`|
|M9|个人中心|个人信息查看与编辑|`/profile`|
|M10|管理员后台|用户列表管理、登录日志、平台数据统计|`/admin/*`|
|—|布局壳|侧边栏 + 顶栏 + 全局提醒弹窗|所有业务路由|

## 2. M1 用户认证模块

### 2.1 功能描述

提供登录和注册两个独立页面，是用户进入系统的入口。

### 2.2 页面结构

```Mermaid
graph TD
    LOGIN[LoginView.vue] --> FORM1[登录表单]
    FORM1 --> PHONE1[手机号输入]
    FORM1 --> PWD1[密码输入]
    FORM1 --> BTN1[登录按钮]
    LOGIN --> LINK1["跳转注册页"]

    REG[RegisterView.vue] --> FORM2[注册表单]
    FORM2 --> PHONE2[手机号输入]
    FORM2 --> NAME2[用户名输入]
    FORM2 --> PWD2[密码输入]
    FORM2 --> PWD3[确认密码]
    FORM2 --> BTN2[注册按钮]
    REG --> LINK2["跳转登录页"]
```

### 2.3 表单验证规则

|字段|规则|
|-|-|
|手机号|必填，正则 `/^1[3-9]\d{9}$/`，11 位手机号格式|
|用户名|必填，长度 2-20 字符|
|密码|必填，长度 ≥6 位|
|确认密码|必填，与密码字段一致（自定义 validator）|

### 2.4 交互流程

```Mermaid
sequenceDiagram
    participant U as 用户
    participant V as LoginView
    participant API as auth.ts
    participant Store as userStore
    participant LS as localStorage
    participant R as Router

    U->>V: 输入手机号、密码，点击登录
    V->>V: 表单验证 (validate)
    V->>API: login(phone, password)
    API-->>V: { token, user }
    V->>Store: setLoginInfo({ token, user })
    Store->>LS: 写入 token + userInfo
    V->>R: router.push('/home')
```

**Mock 模式：** 当 `VITE_MOCK_AUTH=true` 时，`auth.ts` 返回模拟 token 和用户数据，延迟 300ms 模拟网络请求。

### 2.5 关键代码结构

```Plain Text
api/auth.ts
├── login(phone, password)    → POST /api/auth/login
└── register(phone, username, password, confirmPassword) → POST /api/auth/register

stores/user.ts
├── token: Ref<string>
├── userInfo: Ref<User | null>
├── isLoggedIn: Computed<boolean>
├── isAdmin: Computed<boolean>
├── setLoginInfo(data)        # 写入 token 和用户信息
└── logout()                  # 清除状态
```

## 4. M3 健康数据管理模块

### 4.1 功能描述

健康数据的**核心 CRUD 模块**，支持手动录入、列表查询、删除和 CSV 批量导入。

### 4.2 数据模型

|字段|类型|说明|
|-|-|-|
|`dataId`|number|记录主键|
|`dataType`|string|数据类型枚举|
|`dataValue`|string|数值（字符串存储以兼容精度）|
|`unit`|string|单位（根据 dataType 自动匹配）|
|`recordTime`|string|记录时间（yyyy-MM-dd HH:mm:ss）|

**数据类型枚举：**

|类型值|中文标签|单位|
|-|-|-|
|`BLOOD_PRESSURE`|血压|mmHg|
|`BLOOD_GLUCOSE`|血糖|mmol/L|
|`WEIGHT`|体重|kg|
|`EXERCISE_MINUTES`|运动时长|分钟|
|`SLEEP_HOURS`|睡眠时长|小时|

### 4.3 交互流程

```Mermaid
stateDiagram-v2
    [*] --> 列表空状态
    列表空状态 --> 录入数据: 填写表单 + 提交
    录入数据 --> 列表展示: 刷新列表
    录入数据 --> 重复确认: 同类型同时间重复
    重复确认 --> 列表展示: 确认覆盖后刷新列表
    列表展示 --> 筛选: 选择类型过滤
    筛选 --> 列表展示: 刷新列表
    列表展示 --> 删除: 确认删除
    删除 --> 列表展示: 刷新列表
    列表展示 --> 批量导入: 选择 CSV 文件
    批量导入 --> 导入冲突处理: 发现重复摘要
    导入冲突处理 --> 列表展示: 覆盖全部或跳过重复后刷新列表
    批量导入 --> 列表展示: 刷新列表
```

### 4.4 CSV 批量导入

- 通过 `<input type="file" accept=".csv">` 选择文件

- 使用 `FormData` 上传，设置 `Content-Type: multipart/form-data`

- 后端解析后返回 `{ importedCount, overwrittenCount, skippedDuplicateCount }`

- 若后端返回重复摘要，前端展示统一处理弹窗，支持“覆盖全部”或“跳过重复”

### 5.3 Canvas 绘制逻辑

```TypeScript
// 绘制步骤
1. 获取 canvas 2D context，计算 DPR 适配高清屏
2. 绘制白色背景
3. 绘制水平网格线 + Y 轴标签（4 条等分线）
4. 绘制 X 轴日期标签（等距采样，避免拥挤）

```

### 5.4 统计指标

|指标|计算|
|-|-|
|平均值|`sum(values) / length`|
|最高值|`Math.max(...values)`|
|最低值|`Math.min(...values)`|
|趋势描述|首尾值对比：上升/下降/稳定 + 变化百分比|

```Plain Text
│       └── 分页

```

### 8.3 已读状态管理

```Mermaid
flowchart TD
    CLICK[用户点击推送条目] --> CHECK_READ{isRead === 1?}
    CHECK_READ -->|是| SKIP[跳过]
    CHECK_READ -->|否| SET_READ["item.isRead = 1"]
    SET_READ --> SAVE_LS["localStorage 持久化已读 ID"]
    SAVE_LS --> CALL_API["调用 markPushRead API"]
    CALL_API --> SHOW[展示详情弹窗]
    SKIP --> SHOW

    INIT[列表加载] --> READ_IDS["读取 localStorage 已读 ID Set"]
    READ_IDS --> MERGE["合并到列表项的 isRead 字段"]
```

**双重标记策略：**

- `localStorage` — 持久化已读推送 ID 集合，修复接口返回的 `isRead` 不准确的问题

- API `markPushRead` — 通知后端标记已读（请求失败不影响前端展示）

### 8.4 知识缓存

- 使用 `Map<knowledgeId, KnowledgeItem>` 在内存中缓存已加载的知识条目

- 推送详情弹窗优先从缓存读取完整内容

- 缓存未命中时异步拉取，避免弹窗内容为空

## 9. M8 健康提醒模块

### 9.1 功能描述

用户可以创建定时提醒（吃药/运动/睡眠/喝水/自定义），系统在设定时间通过全局弹窗推送提醒通知。

### 9.2 核心交互

```Plain Text
ReminderView.vue
├── 提示条 (el-alert)       # 使用说明
├── 新增表单                 # 类型下拉 + 时间选择器 + 添加按钮
├── 提醒列表                 # 表格展示，含开关和编辑按钮
└── 编辑弹窗 (el-dialog)    # 修改类型和时间
```

### 9.3 全局提醒弹窗机制

```Mermaid
sequenceDiagram
    participant Timer as 30秒定时器
    participant Popup as ReminderPopup.vue
    participant API as reminder.ts
    participant U as 用户

    loop 每30秒
        Timer->>API: getReminders() 获取所有提醒
        API-->>Timer: Reminder[]
        Timer->>Timer: 筛选 status=1 且时间匹配 (±1min)
        Timer->>Popup: 设置 reminder + visible=true
        Popup->>U: 右下角弹出提醒卡片
    end

    U->>Popup: 点击「已完成」→ 关闭弹窗
    U->>Popup: 点击「稍后提醒」→ 关闭 + 10分钟后重新检查
    U->>Popup: 点击「✕」→ 关闭弹窗
```

### 9.4 时间匹配算法

```TypeScript
function isTimeMatch(reminderTime: string, currentTime: string): boolean {
  const [h1, m1] = reminderTime.split(':').map(Number)
  const [h2, m2] = currentTime.split(':').map(Number)
  return h1 === h2 && Math.abs(m1 - m2) <= 1
}
```

**关键设计：**

- 30 秒轮询间隔，±1 分钟容差确保不会漏掉提醒

- 已触发的提醒 ID 加入 `checkedReminders` Set，避免重复弹出

- "稍后提醒"清除该 ID 的已检查标记，10 分钟后恢复检查

### 9.5 提醒类型

|类型值|标签|弹窗图标|默认文案|
|-|-|-|-|
|`MEDICATION`|吃药提醒|💊|请按时服药，保持健康从规律用药开始。|
|`EXERCISE`|运动提醒|🏃|运动让身体更健康，一起动起来吧！|
|`SLEEP`|睡眠提醒|😴|充足的睡眠是健康的基石，该休息了。|
|`WATER`|喝水提醒|💧|每天喝足够的水，保持身体水分平衡。|
|`CUSTOM`|自定义|📌|别忘了您设定的提醒事项哦。|

### 12.2 AppHeader 顶栏

- 左侧：系统标题 "AI 健康管理系统"

- 右侧：当前用户名 + 退出登录按钮

- 退出登录：清除 Pinia + localStorage，跳转 `/login`

### 12.3 ReminderPopup 全局弹窗

- 使用 `<Teleport to="body">` 脱离组件树挂载

- 右下角滑入动画（`slideUp`）

- 三种操作：已完成 / 稍后提醒（10分钟） / 关闭

- 30 秒轮询 + 时间匹配触发

### 12.4 LoadingSpinner / EmptyState

- `LoadingSpinner` — 旋转加载图标 + 可选文字提示（如"AI 正在分析..."）

- `EmptyState` — 文档图标 + 可选描述文字

- 均在多个模块中复用
