## 1. 引言

### 1.1 项目背景

随着居民健康意识提升，传统医疗模式逐步向**主动健康管理**转型。个人日常健康数据（血压、血糖、运动、睡眠等）缺乏系统化管理与专业分析指导。本项目为软件工程课程实践项目，旨在开发一套轻量化AI个性化私人健康管理系统，帮助普通用户实现健康数据管理、AI健康分析、个性化健康指导与风险预警，简化复杂技术实现，聚焦软件工程全流程落地。

### 1.2 项目目标

1. 开发Web端健康管理系统，覆盖**用户管理、健康数据录入、AI健康分析、健康提醒、数据可视化、管理员后台**核心功能。

2. 基于前后端分离架构，集成Spring Boot、Vue、FastAPI与公开大模型API，保证系统可运行、功能闭环、可演示。

3. 严格遵循软件工程流程，完成需求、设计、开发、测试、部署全环节，满足课程考核要求。

4. AI功能仅提供生活方式健康指导，**不涉及医疗诊断、用药推荐**，确保合规性。

### 1.3 项目范围

#### 1.3.1 包含范围

- 用户端：注册登录、个人信息管理、健康数据手动录入/批量导入、AI健康画像、个性化健康计划、健康提醒、健康趋势可视化、健康知识推送。

- 管理员端：用户管理、健康数据统计、系统基础配置、咨询记录查看。

- 服务端：用户权限控制、数据存储与处理、AI服务对接、定时提醒任务、接口管理。

#### 1.3.2 排除范围

- 不接入真实可穿戴设备数据，仅支持手动录入与模拟数据导入。

- 不对接医疗机构、医保系统，仅做健康管理，无诊疗、挂号功能。

- 不开发移动端App，仅实现Web端适配，放弃复杂跨平台兼容。

- 不实现复杂医疗AI模型训练，仅调用公开大模型API，轻量化实现AI功能。

### 1.4 读者对象

本文档面向项目开发团队、测试团队、课程考核评审组，用于指导项目设计、开发、测试与验收工作。

### 1.5 文档约定

- 本文档优先级：Must（必须实现）> Should（应该实现）> Could（可选实现）。

- 技术术语：

    - **AI服务**：基于FastAPI封装的大模型接口，提供健康分析与建议生成能力。

    - **健康数据**：血压、血糖、体重、运动时长、睡眠时长等个人日常健康指标。

## 2. 总体描述

### 2.1 产品愿景

打造轻量化、易操作、合规化的个人健康管理工具，让普通用户无需专业医疗知识，即可通过系统完成健康数据管理、AI健康评估与个性化健康指导，助力养成健康生活习惯。

### 2.2 用户群体

#### 2.2.1 普通用户（核心用户）

- 特征：普通健康人群、亚健康人群，年龄18-65岁，无专业医疗背景，关注日常健康管理。

- 核心诉求：记录健康数据、获取AI健康分析、接收健康提醒、查看健康趋势、学习健康知识。

#### 2.2.2 管理员

- 特征：系统维护人员，负责用户管理、数据统计、系统基础配置。

- 核心诉求：查看用户列表、管理用户账号、统计平台健康数据、监控系统运行状态。

### 2.3 产品功能概述

系统采用**前后端分离架构**，整体分为三层：

1. **前端层**：Vue实现用户端与管理员端界面，负责交互展示、数据录入、可视化呈现。

2. **业务服务层**：Spring Boot实现核心业务逻辑，处理数据存储、权限控制、定时任务，对接AI服务。

3. **AI服务层**：FastAPI封装公开大模型API，实现健康画像、健康计划生成、健康知识推送。

### 2.4 运行环境

- 操作系统：Windows 10及以上

- 浏览器：Chrome、Firefox、Edge（主流浏览器）

- 服务端：Spring Boot 2.7+、FastAPI 0.95+

- 数据库：MySQL 8.0

- 依赖环境：JDK 1.8+、Python 3.9+、Node.js 16+

## 3. 功能需求

### 3.1 用户管理模块（Must）

#### 3.1.1 用户注册

- 功能描述：用户输入手机号、用户名、密码完成注册，支持手机号唯一性校验。

- 输入：手机号、用户名、密码、确认密码

- 输出：注册成功提示/手机号已存在/密码不一致提示

#### 3.1.2 用户登录

- 功能描述：用户输入手机号+密码登录 / 找回系统，支持身份校验。

- 输入：手机号、密码

- 输出：登录成功（跳转首页）/账号密码错误提示

#### 3.1.3 个人信息以及账号管理

- 功能描述：用户查看/修改个人基础信息（姓名、年龄、性别、身高、联系方式）或密码，支持信息保存。

- 输入：修改后的个人信息或密码

- 输出：信息修改成功提示

### 3.2 健康数据管理模块（Must）

#### 3.2.1 健康数据手动录入

- 功能描述：用户手动录入日常健康数据，包括：血压、血糖、体重、运动时长、睡眠时长，支持提交保存。

- 输入：数据类型、数值、录入时间

- 输出：数据保存成功提示、录入记录列表

#### 3.2.2 健康数据批量导入

- 功能描述：用户上传CSV格式模拟数据文件（提供可下载模板），批量导入历史健康数据。

- 输入：CSV文件

- 输出：导入成功提示、导入数据列表、导入失败提示

#### 3.2.3 健康数据查询

- 功能描述：用户按时间范围、数据类型查询历史健康数据，支持分页展示。

- 输入：开始时间、结束时间、数据类型

- 输出：符合条件的健康数据列表

### 3.3 AI健康分析模块（Must）

#### 3.3.1 AI健康画像生成

- 功能描述：系统基于用户历史健康数据，调用AI服务生成健康画像，标注健康标签（如“血压正常”“睡眠不足”）与风险等级（低/中/高）。

- 输入：用户ID、历史健康数据

- 输出：健康画像报告（标签、风险等级、简要分析）

#### 3.3.2 个性化健康计划生成

- 功能描述：用户提交健康需求（如“减脂”“控血压”），AI服务生成饮食、运动、睡眠三类个性化健康建议。

- 输入：用户ID、健康需求

- 输出：个性化健康计划（饮食建议、运动建议、睡眠建议）

#### 3.3.3 健康知识推送

- 功能描述：系统根据用户健康画像，自动推送匹配的健康科普知识（如高血压饮食指南、运动常识）。

- 输入：用户ID、健康标签

- 输出：健康知识列表

### 3.4 健康提醒模块（Must）

#### 3.4.1 定时提醒设置

- 功能描述：用户设置提醒类型（数据录入、运动、作息）、提醒时间，开启/关闭提醒。

- 输入：提醒类型、提醒时间、状态

- 输出：提醒设置成功提示、提醒列表

#### 3.4.2 提醒推送

- 功能描述：到达设定时间，系统前端弹窗推送提醒信息。

- 输入：无

- 输出：弹窗提醒（提醒内容、关闭按钮）

### 3.5 健康可视化模块（Must）

#### 3.5.1 健康趋势图展示

- 功能描述：系统基于用户历史数据，生成可视化图表，包括：体重/血压趋势折线图、运动/睡眠占比饼图。

- 输入：用户ID、时间范围、数据类型

- 输出：可视化图表、图表下载功能

#### 3.5.2 健康报告导出

- 功能描述：用户可导出指定时间范围的健康报告（含数据列表、趋势图、AI分析结论），支持PDF格式。

- 输入：用户ID、时间范围

- 输出：PDF格式健康报告

### 3.6 管理员管理模块（Must）

#### 3.6.1 用户管理

- 功能描述：管理员查看所有用户列表，支持查询、禁用/启用用户账号。

- 输入：用户名、手机号、账号状态

- 输出：用户列表、操作成功提示

#### 3.6.2 健康数据统计

- 功能描述：管理员查看平台整体健康数据统计（用户数量、数据录入量、健康风险分布），支持图表展示。

- 输入：时间范围

- 输出：统计图表、统计数据列表

## 4. 非功能需求

### 4.1 性能需求

1. 页面加载响应时间≤2秒，数据查询响应时间≤1秒。

2. 单个用户连续快速操作时界面响应正常，系统无卡顿、无数据丢失。

3. AI接口响应时间≤5秒，高峰期无超时报错。

### 4.2 安全性需求

1. 用户密码采用加密存储，禁止明文存储。

2. 用户敏感数据（手机号、个人信息）脱敏展示，隐藏中间字符。

3. 角色权限隔离：普通用户仅查看个人数据，管理员无用户隐私数据查看权限。

4. 接口请求校验，防止非法请求、SQL注入攻击。

### 4.3 易用性需求

1. 界面简洁清晰，操作流程≤3步完成核心操作（数据录入、查询）。

2. 按钮、提示文字通俗易懂，无专业术语，适配普通用户认知。

3. 页面布局合理，主流浏览器适配，无排版错乱问题。

### 4.4 可靠性需求

1. 系统本地运行稳定，连续运行72小时无崩溃、无异常退出。

2. 数据提交后自动保存，网络中断恢复后数据不丢失。

3. AI服务调用失败时，返回友好提示，不影响系统其他功能使用。

### 4.5 可维护性需求

1. 代码模块化、注释清晰，符合开发规范，便于后续修改扩展。

2. 接口文档完整，标注参数、返回值、调用方式。

3. 数据库表设计合理，字段命名规范，便于数据维护。

### 4.6 合规性需求

1. AI健康建议仅提供生活方式指导，**无医疗诊断、用药推荐、疾病治疗相关内容**。

2. 健康知识内容符合《中国居民膳食指南》《健康生活常识》规范，无虚假、违规信息。

3. 用户数据仅用于个人健康管理，不泄露、不对外共享，符合隐私保护要求。

## 5. 接口需求

### 5.1 前端-后端接口

- 用户接口：注册接口、登录接口、个人信息查询/修改接口。

- 健康数据接口：数据录入接口、数据查询接口、批量导入接口。

- AI接口：健康画像生成接口、健康计划生成接口、健康知识推送接口。

- 提醒接口：提醒设置接口、提醒查询接口。

- 可视化接口：趋势图数据接口、健康报告导出接口。

- 管理员接口：用户管理接口、数据统计接口、咨询记录接口。

### 5.2 后端-AI服务接口

- 接口协议：HTTP/HTTPS

- 请求方式：POST

- 接口地址：[http://localhost:8000/api/ai/xxx](http://localhost:8000/api/ai/xxx)

- 输入参数：用户ID、健康数据、prompt指令

- 输出参数：JSON格式，包含分析结果、健康建议、状态码、提示信息

### 5.3 数据库接口

- 数据库类型：MySQL 8.0

- 连接方式：JDBC

- 核心表：用户表、健康数据表、提醒表、AI咨询记录表
