# 个人信息管理前端实现说明

本文档记录个人信息管理页面第一版的前端实现内容，便于前后端联调、测试人员编写测试用例。

## 1. 实现范围

本次实现基于 `docs/api/后端接口草案.md` 中的用户接口：

- 个人信息展示：调用 `GET /api/users/me` 获取当前用户信息，以卡片形式展示。
- 个人信息编辑：提供编辑表单，可修改用户名、出生日期、性别、身高。
- 编辑暂用 Mock 模式（`VITE_MOCK_EDIT=true`），等后端 `PUT /api/users/me` 就绪后移除 Mock 即可。

以下字段为只读，不开放编辑：

- 手机号（后端已脱敏，如 `138****8000`）
- 角色（USER / ADMIN）
- 用户 ID

## 2. 前端分层

本次个人信息模块按 Vue 3 常见分层实现：

- `views/ProfileView.vue`：页面组件，管理展示/编辑两种模式的切换。
- `api/user.ts`：封装 `GET /api/users/me` 和 `PUT /api/users/me` 请求，含 Mock 开关。
- `stores/user.ts`：Pinia store，管理登录状态和用户信息，编辑保存后同步更新 store。
- `types/user.ts`：TypeScript 类型定义，与后端 `CurrentUserResponse` 字段对齐。

## 4. 类型定义对齐

前端 `User` 类型与后端 `CurrentUserResponse` 字段对应关系：

| 前端字段 | 后端字段 | 类型 | 可编辑 |
|----------|----------|------|--------|
| `userId` | `userId` | `number` | 否 |
| `username` | `username` | `string` | 是 |
| `phone` | `phone`（已脱敏） | `string` | 否 |
| `birthDate` | `birthDate` | `string` | 是 |
| `gender` | `gender` | `'MALE' \| 'FEMALE' \| 'UNKNOWN'` | 是 |
| `height` | `height` | `number` | 是 |
| `role` | `role` | `'USER' \| 'ADMIN'` | 否 |

后端 `gender` 枚举值为 `MALE` / `FEMALE` / `UNKNOWN`，前端下拉框映射为 男 / 女 / 保密。

## 5. Mock 编辑模式

编辑提交时，若 `.env` 中设置 `VITE_MOCK_EDIT=true`（默认开启），前端模拟成功返回，不真实调用后端：

```typescript
if (import.meta.env.VITE_MOCK_EDIT === 'true') {
  await new Promise((r) => setTimeout(r, 300))
  return { data: { code: 0, message: '修改成功', data: null } }
}
return request.put('/api/users/me', data)
```

后端 `PUT /api/users/me` 实现后，将 `.env` 中 `VITE_MOCK_EDIT` 设为 `false` 或删除即可切换为真实调用。

## 6. 交互流程

1. 页面加载 → 调 `GET /api/users/me` → 渲染展示卡片
2. 用户点击「编辑」→ 表单预填当前值，进入编辑模式
3. 用户修改字段 → 点击「保存」→ 调 `PUT /api/users/me`（或 Mock）→ 成功后刷新展示 + 同步 Pinia store
4. 用户点击「取消」→ 丢弃修改，回到展示模式
5. 请求失败 → Element Plus Message 提示错误信息

## 7. 响应格式

与后端统一响应格式一致：

```json
    "height": 175.5,
    "role": "USER"
  }
}
```
