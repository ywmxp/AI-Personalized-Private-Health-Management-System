# 健康数据管理前端实现说明

本文档记录健康数据管理页面第一版的前端实现内容，便于前后端联调、测试人员编写测试用例。

## 1. 实现范围

本次实现基于 `docs/api/后端接口草案.md` 中的健康数据接口（§3）：

- 健康数据手动录入：调用 `POST /api/health-data`。
- 健康数据查询：调用 `GET /api/health-data`，支持按数据类型、时间范围筛选和分页。
- 健康数据删除：调用 `DELETE /api/health-data/{dataId}`。
- 健康数据批量导入（Should）：调用 `POST /api/health-data/import`，上传 CSV 文件。
- 健康趋势查询：调用 `GET /api/health-data/trends`。

暂不实现：健康数据修改（API 未定义 PUT 接口）、健康报告导出。

## 2. 前端分层

- `views/HealthDataView.vue`：页面组件，Tab「数据管理」+ Tab「批量导入」。
- `api/health.ts`：封装健康数据 5 个 HTTP 请求，无 Mock，直接调用后端。
- `types/health.ts`：TypeScript 类型定义，与 API 文档 §3 字段对齐。

## 4. 类型定义

### 录入请求（API §3.1）

| 字段 | 类型 | 说明 |
|------|------|------|
| `dataType` | `string` | BLOOD_PRESSURE / BLOOD_GLUCOSE / WEIGHT / EXERCISE_MINUTES / SLEEP_HOURS |
| `dataValue` | `string` | 数值，string 类型兼容血压 `120/80` |
| `unit` | `string` | 单位 |
| `recordTime` | `string` | `yyyy-MM-dd HH:mm:ss` |
| `overwrite` | `boolean` | 发现重复记录时确认覆盖 |

### 列表项（API §3.2）

| 字段 | 类型 | 说明 |
|------|------|------|
| `dataId` | `number` | 主键 |
| `dataType` | `string` | 数据类型 |
| `dataValue` | `string` | 数值 |
| `unit` | `string` | 单位 |
| `recordTime` | `string` | 记录时间 |

### 分页格式

```json
{ "items": [], "page": 1, "size": 10, "total": 0 }
```

## 5. API 模块

| 函数 | HTTP | 端点 |
|------|------|------|
| `createRecord(data)` | POST | `/api/health-data` |
| `queryRecords(params)` | GET | `/api/health-data` |
| `deleteRecord(dataId)` | DELETE | `/api/health-data/{dataId}` |
| `importCsv(file)` | POST | `/api/health-data/import` |
| `getTrends(params)` | GET | `/api/health-data/trends` |

所有函数直接调用后端，无 Mock 模式。

补充：

- `createRecord(data)` 在发现同类型同时间重复数据时，会收到后端返回的冲突详情，前端弹出对比确认框后再带 `overwrite=true` 二次提交。
- `importCsv(file)` 在发现批量重复数据时，会收到重复摘要，前端展示“覆盖全部 / 跳过重复”选择，不逐条弹窗。

## 7. 交互流程

### 录入
填写表单 → 校验通过 → `createRecord()`

- 若无重复：成功后保留当前表单输入并刷新列表
- 若发现重复：展示“已有数据 / 当前录入”对比弹窗 → 用户确认后带 `overwrite=true` 再次提交 → 刷新列表

### 查询
选择筛选项 → 点击查询（或录入成功后自动触发） → `queryRecords()` → 渲染表格

补充：

- 列表排序由后端统一按 `recordTime` 倒序返回，前端直接渲染。
- 数据类型筛选与时间范围筛选可以组合使用。

### 删除
点击删除 → `ElMessageBox.confirm` 确认 → `deleteRecord()` → 刷新列表

### 批量导入
下载模板 → 填写 CSV → 选择文件 → 点击导入 → `importCsv()`（multipart/form-data）

- 若无重复：直接提示导入成功并刷新列表
- 若发现重复：展示重复摘要弹窗 → 用户选择“覆盖全部”或“跳过重复” → 携带批量处理参数重新提交

- 分页支持切换每页条数（10/20/50）。
