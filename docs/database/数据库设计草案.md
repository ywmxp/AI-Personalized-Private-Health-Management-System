# 数据库设计草案 v1

## 1. 设计约定

- 主键统一使用 `BIGINT`。
- 表名使用小写下划线。
- 时间字段使用 `DATETIME`。
- 状态、角色、类型字段使用英文枚举值，前端展示时再转换为中文。
- 密码字段只存储加密后的密码。
- 手机号在接口返回时需要脱敏。


## 2. 表结构草案

### 2.1 用户表 `user`

|字段|类型|约束|说明|
|-|-|-|-|
|user_id|BIGINT|PK, AUTO_INCREMENT|用户 ID|
|username|VARCHAR(50)|NOT NULL|用户名|
|phone|VARCHAR(20)|NOT NULL, UNIQUE|手机号|
|password_hash|VARCHAR(100)|NOT NULL|加密密码|
|birth_date|DATE|NULL|出生日期|
|gender|VARCHAR(20)|NULL|性别：MALE / FEMALE / UNKNOWN|
|height|DECIMAL(5,2)|NULL|身高，单位 cm|
|role|VARCHAR(20)|NOT NULL|角色：USER / ADMIN|
|status|VARCHAR(20)|NOT NULL|状态：ENABLED / DISABLED|
|create_time|DATETIME|NOT NULL|创建时间|
|update_time|DATETIME|NOT NULL|更新时间|

索引：

- `uk_user_phone(phone)`
- `idx_user_role(role)`
- `idx_user_status(status)`

管理员账号策略：

- 普通注册接口固定创建 `USER` 角色。
- 通过初始化 SQL 创建一个默认 `ADMIN` 账号。
- 不开放公开的管理员注册接口。
- 后续如果需要新增管理员，应由已登录的管理员通过后台接口创建。

### 2.2 健康数据表 `health_data`

|字段|类型|约束|说明|
|-|-|-|-|
|data_id|BIGINT|PK, AUTO_INCREMENT|健康数据 ID|
|user_id|BIGINT|NOT NULL|用户 ID|
|data_type|VARCHAR(30)|NOT NULL|数据类型|
|data_value|VARCHAR(50)|NOT NULL|数据值|
|unit|VARCHAR(20)|NULL|单位|
|record_time|DATETIME|NOT NULL|记录时间|
|create_time|DATETIME|NOT NULL|创建时间|
|update_time|DATETIME|NOT NULL|更新时间|

第一版数据类型：

- `BLOOD_PRESSURE`
- `BLOOD_GLUCOSE`
- `WEIGHT`
- `EXERCISE_MINUTES`
- `SLEEP_HOURS`

说明：

- 第一版把 `data_value` 作为字符串，便于兼容血压 `120/80`。
- 若后续图表统计复杂，再拆出数值字段。

索引：

- `idx_health_data_user_time(user_id, record_time)`
- `idx_health_data_user_type_time(user_id, data_type, record_time)`
