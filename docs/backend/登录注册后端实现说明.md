# 登录注册后端实现说明

本文档记录登录注册后端第一版的主要实现内容，便于前端联调、后端组维护和后续测试人员编写测试用例。

## 1. 实现范围

本次实现基于 `docs/api/后端接口草案.md` 中的认证接口：

- `POST /api/auth/register`：普通用户注册。
- `POST /api/auth/login`：用户登录，成功后返回 JWT token。
- `GET /api/users/me`：根据请求头中的 token 查询当前登录用户信息。

第一阶段不开放管理员注册接口。普通注册接口固定创建 `USER` 角色，账号状态默认为 `ENABLED`。

## 2. 后端分层

本次认证模块按 Spring Boot 常见分层实现：

- `controller`：接收 HTTP 请求，返回统一响应。
- `service`：处理注册、登录、当前用户查询等业务逻辑。
- `repository`：通过 Spring Data JPA 操作数据库。
- `domain`：数据库表对应的实体类。
- `dto`：前后端接口请求和响应对象。
- `security`：JWT 生成、解析和请求鉴权过滤器。
- `exception`：统一业务异常和参数校验异常处理。

## 3. 数据库表

新增初始化 SQL：

```text
docker/mysql/init/001_auth_tables.sql
```

包含两张表：

- `user`：保存用户账号、手机号、加密密码、角色、状态等信息。
- `login_log`：保存登录成功和失败记录，便于后续管理员查询登录日志。

如果本地 MySQL 容器是旧数据卷，新增初始化 SQL 不会自动重新执行，需要手动导入或重建数据库卷。新同学首次启动容器时会自动执行该 SQL。

## 4. 安全与配置

密码使用 Spring Security 的 `BCryptPasswordEncoder` 加密保存，只存储 `password_hash`，不保存明文密码。

登录成功后后端生成 JWT，前端后续访问受保护接口时需要携带：

```http
Authorization: Bearer <token>
```

已在配置模板中补充：

- `JWT_SECRET`
- `JWT_EXPIRE_MINUTES`

本地开发时默认放行：

- `/api/auth/register`
- `/api/auth/login`
- `/api/health`
- `/actuator/health`

其他接口默认需要登录 token。

## 5. 响应格式

接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

注册成功示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": 1
  }
}
```

登录成功示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "user": {
      "userId": 1,
      "username": "test_user",
      "phone": "139****0001",
      "role": "USER"
    }
  }
}
```

## 6. 已验证内容

本次本地验证通过：

- `./mvnw test` 构建测试通过。
- Docker MySQL 正常启动。
- `POST /api/auth/register` 注册成功。
- 重复手机号注册返回错误。
- `POST /api/auth/login` 登录成功并返回 token。
- 不带 token 请求 `/api/users/me` 返回 `401`。
- 带 token 请求 `/api/users/me` 返回当前用户信息。
- 错误密码登录返回错误，并写入 `login_log` 失败记录。

默认 Spring 测试不依赖真实数据库，只用于保证项目能编译并加载基础上下文。真实数据库联调通过 Docker MySQL 和接口手动验证完成。
