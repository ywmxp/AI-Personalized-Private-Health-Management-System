# 健康数据 CSV 导入后端实现说明

本文档记录健康数据 CSV 批量导入接口的后端实现约定，便于前端联调、后端维护和测试用例编写。

## 1. 实现范围

本次实现基于 `docs/api/后端接口草案.md` 中的健康数据批量导入接口：

- `POST /api/health-data/import`：上传 CSV 文件并批量写入当前登录用户的健康数据。

CSV 模板当前由前端静态文件提供：

```text
frontend/public/health_data_template.csv
```

前端下载路径为：

```http
GET /health_data_template.csv
```

后端不再额外提供模板下载接口，避免与当前前端静态模板方案冲突。

## 2. 接口约定

请求：

```http
POST /api/health-data/import
Content-Type: multipart/form-data
Authorization: Bearer <token>
```

表单字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | CSV 文件 | 是 | 健康数据导入文件 |

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "importedCount": 5
  }
}
```

说明：

- 成功响应统一使用 `ApiResponse.success(...)`，业务成功码固定为 `code = 0`。
- `importedCount` 表示本次实际写入的健康数据记录数。
- 接口需要登录，导入数据归属 JWT 中的当前用户。

## 3. CSV 格式

CSV 表头固定为：

```csv
dataType,dataValue,unit,recordTime
```

示例：

```csv
dataType,dataValue,unit,recordTime
WEIGHT,70.5,kg,2026-06-01 08:30:00
BLOOD_PRESSURE,120/80,mmHg,2026-06-01 09:00:00
BLOOD_GLUCOSE,5.2,mmol/L,2026-06-01 07:30:00
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `dataType` | 是 | 健康数据类型，如 `WEIGHT`、`BLOOD_PRESSURE` |
| `dataValue` | 是 | 健康数据值，按字符串保存 |
| `unit` | 否 | 单位，如 `kg`、`mmHg`、`mmol/L` |
| `recordTime` | 是 | 记录时间，格式固定为 `yyyy-MM-dd HH:mm:ss` |

## 4. 后端处理逻辑

实现位置：

- `HealthDataController#importCsv`
- `HealthDataService#importCsv`

处理流程：

1. Controller 接收 `multipart/form-data` 中的 `file`。
2. Service 校验文件是否为空。
3. 使用 UTF-8 读取 CSV，支持 UTF-8 BOM 表头。
4. 校验表头必须完全匹配 `dataType,dataValue,unit,recordTime`。
5. 跳过空行，逐行校验并转换为 `HealthData` 实体。
6. 使用 `healthDataRepository.saveAll(...)` 批量保存。
7. 返回保存数量 `importedCount`。

事务策略：

- `importCsv` 使用 `@Transactional`。
- 任一数据行格式错误时抛出 `BusinessException(ErrorCode.VALIDATION_ERROR, ...)`。
- 出错时整文件导入失败，不写入部分数据。

当前 CSV 解析能力：

- 支持普通逗号分隔格式。
- 支持空 `unit` 字段。
- 支持 UTF-8 BOM。
- 不支持字段内逗号、复杂引号转义等高级 CSV 语法；模板中也不包含这类场景。

## 5. 错误处理

常见错误：

| 场景 | 返回 code | message 示例 |
|---|---:|---|
| 未上传文件或空文件 | `40000` | `CSV文件不能为空` |
| 表头错误 | `40000` | `CSV表头必须为dataType,dataValue,unit,recordTime` |
| 列数错误 | `40000` | `第2行：列数必须为4` |
| 必填字段为空 | `40000` | `第2行：dataValue不能为空` |
| 时间格式错误 | `40000` | `第2行：recordTime格式必须为yyyy-MM-dd HH:mm:ss` |
| 未登录 | `40100` | `请先登录` |

错误响应仍沿用全局异常格式：

```json
{
  "code": 40000,
  "message": "第2行：recordTime格式必须为yyyy-MM-dd HH:mm:ss",
  "data": null
}
```
