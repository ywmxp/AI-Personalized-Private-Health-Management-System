# 健康数据录入与导入后端实现说明

本文档记录健康数据模块当前后端实现约定，覆盖单条录入、列表查询默认排序、CSV 批量导入及重复数据处理，便于前端联调、后端维护和测试用例编写。

当前实现共用同一套健康数据业务校验规则，核心涉及：

- `POST /api/health-data`
- `GET /api/health-data`
- `POST /api/health-data/import`

## 1. 实现范围

本次实现基于 `docs/api/后端接口草案.md` 中的健康数据接口：

- `POST /api/health-data`：新增单条健康数据，支持重复冲突确认后覆盖。
- `GET /api/health-data`：分页查询健康数据，支持按类型、时间范围筛选，默认按记录时间倒序返回。
- `POST /api/health-data/import`：上传 CSV 文件并批量写入当前登录用户的健康数据，支持“覆盖重复”或“跳过重复”。

CSV 模板当前由前端静态文件提供：

```text
frontend/public/health_data_template.csv
```

前端下载路径为：

```http
GET /health_data_template.csv
```

后端不再额外提供模板下载接口，避免与当前前端静态模板方案冲突。

## 2. 接口约定

### 2.1 单条录入

请求：

```http
POST /api/health-data
Content-Type: application/json
Authorization: Bearer <token>
```

请求体：

```json
{
  "dataType": "WEIGHT",
  "dataValue": "70.5",
  "unit": "kg",
  "recordTime": "2026-06-01 08:30:00",
  "overwrite": false
}
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `dataType` | 是 | 健康数据类型 |
| `dataValue` | 是 | 健康数据值，按字符串保存 |
| `unit` | 否 | 单位；为空时由后端补默认单位 |
| `recordTime` | 否 | 记录时间；为空时后端默认写入当前时间 |
| `overwrite` | 否 | 是否覆盖同时间同类型已有数据；默认 `false` |

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "dataId": 1
  }
}
```

重复冲突响应：

```json
{
  "code": 40901,
  "message": "该时间已存在同类型健康数据，请确认是否覆盖原记录",
  "data": {
    "existingRecord": {
      "dataType": "WEIGHT",
      "dataValue": "71.0",
      "unit": "kg",
      "recordTime": "2026-06-01 08:30:00"
    },
    "incomingRecord": {
      "dataType": "WEIGHT",
      "dataValue": "70.5",
      "unit": "kg",
      "recordTime": "2026-06-01 08:30:00"
    }
  }
}
```

重复判定口径：

- 同一用户
- 同一 `dataType`
- 同一 `recordTime`

覆盖行为：

- 当前端二次提交且 `overwrite=true` 时，后端更新已有记录的 `dataValue` 和 `unit`。
- 覆盖后保留原记录 `dataId` 和 `recordTime`，仅更新内容与 `updateTime`。

### 2.2 查询接口

请求：

```http
GET /api/health-data?dataType=WEIGHT&startTime=2026-05-01 00:00:00&endTime=2026-05-24 23:59:59&page=1&size=10
Authorization: Bearer <token>
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "dataId": 2,
        "dataType": "WEIGHT",
        "dataValue": "69.8",
        "unit": "kg",
        "recordTime": "2026-06-03 08:30:00"
      },
      {
        "dataId": 1,
        "dataType": "WEIGHT",
        "dataValue": "70.5",
        "unit": "kg",
        "recordTime": "2026-06-01 08:30:00"
      }
    ],
    "page": 1,
    "size": 10,
    "total": 2
  }
}
```

排序规则：

- 默认按 `recordTime DESC`
- 同一 `recordTime` 下再按 `dataId DESC`

### 2.3 CSV 批量导入

请求：

```http
POST /api/health-data/import
Content-Type: multipart/form-data
Authorization: Bearer <token>
```

可选参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `duplicateAction` | `OVERWRITE` / `SKIP` | 否 | 发现重复数据时的批量处理策略 |

表单字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | CSV 文件 | 是 | 健康数据导入文件 |

首次成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "importedCount": 5,
    "overwrittenCount": 0,
    "skippedDuplicateCount": 0
  }
}
```

发现重复且尚未指定处理策略时的响应：

```json
{
  "code": 40902,
  "message": "CSV中存在重复健康数据，请选择覆盖或跳过重复项",
  "data": {
    "duplicateCount": 2,
    "duplicates": [
      {
        "existingRecord": {
          "dataType": "WEIGHT",
          "dataValue": "71.0",
          "unit": "kg",
          "recordTime": "2026-06-01 08:30:00"
        },
        "incomingRecord": {
          "dataType": "WEIGHT",
          "dataValue": "70.5",
          "unit": "kg",
          "recordTime": "2026-06-01 08:30:00"
        }
      }
    ]
  }
}
```

批量处理后的成功响应说明：

- `importedCount`：本次新插入的记录数
- `overwrittenCount`：本次覆盖更新的记录数
- `skippedDuplicateCount`：本次按 `SKIP` 跳过的重复记录数

## 3. CSV 格式

CSV 表头固定为：

```csv
dataType,dataValue,unit,recordTime
```

示例：

```csv
dataType,dataValue,unit,recordTime
WEIGHT,70.5,kg,2026-06-01 08:30:00
BLOOD_PRESSURE,120/80,mmHg,2026-06-01 09:00:00
BLOOD_GLUCOSE,5.2,mmol/L,2026-06-01 07:30:00
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `dataType` | 是 | 健康数据类型，如 `WEIGHT`、`BLOOD_PRESSURE` |
| `dataValue` | 是 | 健康数据值，按字符串保存 |
| `unit` | 否 | 单位，如 `kg`、`mmHg`、`mmol/L` |
| `recordTime` | 是 | 记录时间，格式固定为 `yyyy-MM-dd HH:mm:ss` |

## 4. 后端处理逻辑

实现位置：

- `HealthDataController`
- `HealthDataService`
- `HealthDataRepository`
- `HealthDataValidator`

### 4.1 单条录入处理流程

1. Controller 接收 JSON 请求体。
2. Service 调用共享校验器，规范化 `dataType`、`dataValue`、`unit`、`recordTime`。
3. 按“当前用户 + dataType + recordTime”查询是否已有记录。
4. 若无重复，直接保存。
5. 若有重复且 `overwrite=false`，返回 `40901` 和前端对比所需的 `existingRecord` / `incomingRecord`。
6. 若有重复且 `overwrite=true`，覆盖已有记录的数值和单位。

### 4.2 查询处理流程

1. 根据 `userId` 强制限定数据归属。
2. 按可选条件 `dataType`、`startTime`、`endTime` 组装查询条件。
3. 统一使用分页查询。
4. 返回结果固定按 `recordTime DESC, dataId DESC` 排序。

### 4.3 CSV 导入处理流程

1. Controller 接收 `multipart/form-data` 中的 `file` 和可选 `duplicateAction`。
2. Service 校验文件是否为空。
3. 使用 UTF-8 读取 CSV，支持 UTF-8 BOM 表头。
4. 校验表头必须完全匹配 `dataType,dataValue,unit,recordTime`。
5. 跳过空行，逐行解析 `recordTime` 并调用共享校验器校验业务规则。
6. 先在单个 CSV 内按 `dataType + recordTime` 去重；若同一文件内重复，后出现的记录覆盖前一条待导入记录。
7. 再按“当前用户 + dataType + recordTime”检查数据库中是否已存在重复记录。
8. 若存在重复且 `duplicateAction` 未指定，返回 `40902` 和重复摘要。
9. 若 `duplicateAction=OVERWRITE`，覆盖所有重复项，并插入不冲突的新数据。
10. 若 `duplicateAction=SKIP`，跳过所有重复项，仅插入不冲突的新数据。

事务策略：

- `create` 与 `importCsv` 均使用 `@Transactional`。
- 任一 CSV 数据行格式错误时抛出 `BusinessException(ErrorCode.VALIDATION_ERROR, ...)`。
- CSV 校验错误时整文件导入失败，不写入部分数据。
- CSV 重复冲突不属于校验错误，而是显式返回冲突摘要供前端二次确认。

当前 CSV 解析能力：

- 支持普通逗号分隔格式。
- 支持空 `unit` 字段。
- 支持 UTF-8 BOM。
- 不支持字段内逗号、复杂引号转义等高级 CSV 语法；模板中也不包含这类场景。

共享业务校验规则：

- `dataType` 仅支持 `BLOOD_PRESSURE`、`BLOOD_GLUCOSE`、`WEIGHT`、`EXERCISE_MINUTES`、`SLEEP_HOURS`。
- `BLOOD_PRESSURE`：
  - `dataValue` 必须为 `收缩压/舒张压`，如 `120/80`
  - 收缩压范围 `40-300`
  - 舒张压范围 `30-200`
  - 收缩压必须大于舒张压
- `BLOOD_GLUCOSE`：`dataValue` 必须为 `0.1-50` 之间的数字。
- `WEIGHT`：`dataValue` 必须为 `1-500` 之间的数字。
- `EXERCISE_MINUTES`：`dataValue` 必须为 `0-1440` 之间的整数。
- `SLEEP_HOURS`：`dataValue` 必须为 `0-24` 之间的数字。
- `unit` 允许为空；为空时由后端补默认单位：
  - `BLOOD_PRESSURE` → `mmHg`
  - `BLOOD_GLUCOSE` → `mmol/L`
  - `WEIGHT` → `kg`
  - `EXERCISE_MINUTES` → `分钟`
  - `SLEEP_HOURS` → `小时`
- `unit` 若传入，必须与 `dataType` 匹配。
- `recordTime` 不能晚于服务端当前时间。

单条录入接口与 CSV 导入的差异：

- `POST /api/health-data` 的 `recordTime` 可为空。
- 当 `recordTime` 为空时，后端默认写入当前时间。
- CSV 导入中的 `recordTime` 仍为必填字段。

## 5. 错误处理

常见错误：

| 场景 | 返回 code | message 示例 |
|---|---:|---|
| 未上传文件或空文件 | `40000` | `CSV文件不能为空` |
| 表头错误 | `40000` | `CSV表头必须为dataType,dataValue,unit,recordTime` |
| 列数错误 | `40000` | `第2行：列数必须为4` |
| 必填字段为空 | `40000` | `第2行：dataValue不能为空` |
| 时间格式错误 | `40000` | `第2行：recordTime格式必须为yyyy-MM-dd HH:mm:ss` |
| 不支持的数据类型 | `40000` | `第2行：dataType不支持，必须为BLOOD_PRESSURE,...` |
| 数值格式或范围错误 | `40000` | `第2行：BLOOD_PRESSURE格式必须为收缩压/舒张压，例如120/80` |
| 单位不匹配 | `40000` | `第2行：WEIGHT的unit必须为kg` |
| 未来时间 | `40000` | `第2行：recordTime不能晚于当前时间` |
| 单条录入重复冲突 | `40901` | `该时间已存在同类型健康数据，请确认是否覆盖原记录` |
| CSV 导入重复冲突 | `40902` | `CSV中存在重复健康数据，请选择覆盖或跳过重复项` |
| 未登录 | `40100` | `请先登录` |

错误响应仍沿用全局异常格式：

```json
{
  "code": 40000,
  "message": "第2行：recordTime格式必须为yyyy-MM-dd HH:mm:ss",
  "data": null
}
```
