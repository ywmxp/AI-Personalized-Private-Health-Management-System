# AI 健康分析模块

## 架构概述

**三层职责：**

| 层 | 职责 |
|---|---|
| Spring Boot (8080) | JWT认证、数据库读写、请求/响应格式转换 |
| FastAPI (30000) | Agent管理、Prompt加载、调用DeepSeek大模型 |
| DeepSeek API | 实际AI推理，返回结构化分析结果 |



## 接口调用

> 以下命令使用 PowerShell `Invoke-RestMethod`。端口 `18080` 仅为示例，以你的 `.env` 中 `SERVER_PORT` 为准。

### 1. 登录获取 Token

```powershell
$result = Invoke-RestMethod -Uri http://127.0.0.1:18080/api/auth/login -Method Post -ContentType "application/json" -Body '{"phone":"13800138000","password":"123456"}'
$token = $result.data.token
```

> 如果账号不存在，先调注册接口：
> ```powershell
> Invoke-RestMethod -Uri http://127.0.0.1:18080/api/auth/register -Method Post -ContentType "application/json" -Body '{"phone":"13800138000","username":"test_user","password":"123456","confirmPassword":"123456"}'
> ```

### 2. 生成健康画像

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:18080/api/ai/profile -Method Post -ContentType "application/json" -Headers @{Authorization="Bearer $token"} -Body '{"timeRange":"LAST_7_DAYS"}'
```

| 参数 | 说明 |
|---|---|
| `timeRange` | `LAST_7_DAYS` / `LAST_30_DAYS` / `LAST_90_DAYS` |

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "profileId": 1,
    "healthTags": ["睡眠不足", "运动偏少"],
    "riskLevel": "MEDIUM",
    "analysis": "根据近7天健康数据分析，您的睡眠时长远低于..."
  }
}
```

### 3. 生成健康计划

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:18080/api/ai/plan -Method Post -ContentType "application/json" -Headers @{Authorization="Bearer $token"} -Body '{"healthNeed":"改善睡眠质量,控制体重"}'
```

| 参数 | 说明 |
|---|---|
| `healthNeed` | 健康需求描述，自由文本 |

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "planId": 1,
    "planName": "睡眠改善与体重控制计划",
    "dietSuggest": "建议每日摄入热量控制在1800千卡以内...",
    "sportSuggest": "建议每周进行3-4次有氧运动...",
    "sleepSuggest": "固定每晚22:30就寝，睡前1小时停止使用电子设备..."
  }
}
```

### 4. 健康知识推送

先拿到 `profileId`（从上一步画像接口返回），然后：

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:18080/api/ai/knowledge-push -Method Post -ContentType "application/json" -Headers @{Authorization="Bearer $token"} -Body '{"profileId":1}'
```

| 参数 | 说明 |
|---|---|
| `profileId` | 健康画像ID |

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "pushIds": [1, 2]
  }
}
```

## 错误码

| HTTP状态码 | code | 含义 | 处理 |
|---|---|---|---|
| 401 | 40100 | 未登录 | 重新登录获取token |
| 400 | 40000 | 请求参数不合法 | 检查请求体字段 |
| 503 | — | AI服务未配置 | 检查.env中AI_API_KEY |
| 502 | 50000 | AI服务调用失败 | 检查FastAPI是否启动 |
| 500 | 50000 | 系统异常 | 查看Spring Boot终端日志 |

## 内部数据流详解

### 健康画像生成（POST /api/ai/profile）

```
1. 前端 → Spring Boot
   POST /api/ai/profile
   Body: {"timeRange": "LAST_7_DAYS"}
   Header: Authorization: Bearer <JWT>

2. AiController
   - @AuthenticationPrincipal JwtUser 提取 userId
   - 委托 AiProxyService.generateProfile(userId, "LAST_7_DAYS")

3. AiProxyService
   a) 解析 timeRange → LocalDateTime 起止时间
   b) HealthDataRepository.findByUserIdAndRecordTimeBetween() 查健康数据
   c) 映射为 FastApiHealthDataItem 列表
   d) 组装 FastApiProfileRequest{userId, healthData, timeRange}
   e) RestTemplate.postForObject() → http://127.0.0.1:30000/api/ai/profile
   f) 收到 FastApiProfileResponse{healthTags, riskLevel, analysis}
   g) 创建 HealthProfile 实体，save() 到 health_profile 表
   h) 返回 AiProfileData{profileId, healthTags, riskLevel, analysis}

4. Spring Boot → 前端
   ApiResponse<AiProfileData> JSON
```

### 健康计划生成（POST /api/ai/plan）

```
1. 前端 → Spring Boot
   POST /api/ai/plan
   Body: {"healthNeed": "改善睡眠"}

2. AiProxyService
   a) 查最近30天健康数据
   b) 组装 FastApiPlanRequest → FastAPI POST /api/ai/plan
   c) 收到 FastApiPlanResponse{planName, dietSuggest, sportSuggest, sleepSuggest}
   d) 创建 HealthPlan(is_valid=1, feedback_status="NONE")，save()
   e) 返回 AiPlanData{planId, planName, ...}

3. Spring Boot → 前端
   ApiResponse<AiPlanData> JSON
```

### 健康知识推送（POST /api/ai/knowledge-push）

```
1. 前端 → Spring Boot
   POST /api/ai/knowledge-push
   Body: {"profileId": 1}

2. AiProxyService
   a) HealthProfileRepository.findById(profileId) → 取出 healthTags JSON
   b) KnowledgeRepository.findAll() → 获取全部知识条目
   c) 组装 FastApiPushRequest → FastAPI POST /api/ai/knowledge-push
   d) 收到 FastApiPushResponse{selectedKnowledgeIds, reason}
   e) 遍历 selectedKnowledgeIds，逐一创建 KnowledgePush(is_read=0)，save()
   f) 返回 AiPushData{pushIds}

3. Spring Boot → 前端
   ApiResponse<AiPushData> JSON
```

## FastAPI 内部结构

详见 `backend/src/main/python/` 目录：

```
backend/src/main/python/
├── main.py              # FastAPI 入口
├── config.py            # 读取 .env 中的 AI_API_KEY、AI_MODEL 等
├── prompts/             # Agent Prompt 模板（.md 文件，可直接编辑调试）
│   ├── profile.md       #   健康画像 Prompt
│   ├── plan.md          #   健康计划 Prompt
│   └── push.md          #   知识推送 Prompt
├── agents/              # Agent 封装（每个对应一个 AssistantAgent）
│   ├── base.py          #   Agent 工厂 + Prompt 加载器
│   ├── profile_agent.py #   健康画像 Agent
│   ├── plan_agent.py    #   健康计划 Agent
│   └── push_agent.py    #   知识推送 Agent
├── models/
│   └── schemas.py       #   Pydantic 请求/响应模型
├── services/
│   └── ai_service.py    #   业务编排：格式化数据 → 调用Agent → 解析JSON
└── routers/
    └── ai.py            #   HTTP 路由层：参数校验 → get_ai_service → 委托 service
```

### Prompt 调试

Prompt 文件在 `backend/src/main/python/prompts/` 下，可以直接编辑 `.md` 文件来调整 AI 输出效果（无需重启服务，下次请求生效）。

如果 Agent 返回非 JSON 或格式不对，Spring Boot 会收到 502 错误。

### FastAPI 环境变量

| 变量 | 默认值 | 说明 |
|---|---|---|
| `AI_API_KEY` | (空) | DeepSeek / OpenAI API Key。为空时所有请求返回 503 |
| `AI_API_BASE_URL` | `https://api.openai.com/v1` | API 地址，DeepSeek 用 `https://api.deepseek.com/v1` |
| `AI_MODEL` | `gpt-4o-mini` | 模型名称，DeepSeek 用 `deepseek-v4-pro` |
| `AI_SERVICE_HOST` | `127.0.0.1` | FastAPI 监听地址 |
| `AI_SERVICE_PORT` | `8000` | FastAPI 监听端口 |

## 数据库表

AI 模块涉及 5 张表：

| 表名 | 用途 | 谁写入 |
|---|---|---|
| `health_data` | 用户健康数据记录 | 前端录入 → Spring Boot |
| `health_profile` | AI 生成的健康画像 | Spring Boot（调FastAPI后写入） |
| `health_plan` | AI 生成的健康计划 | Spring Boot（调FastAPI后写入） |
| `knowledge` | 健康知识库 | 管理员 |
| `knowledge_push` | 知识推送记录 | Spring Boot（调FastAPI后写入） |

## 常见问题

| 现象 | 原因 | 解决 |
|---|---|---|
| `AI service not configured` (503) | `.env` 中 `AI_API_KEY` 为空 | 编辑 `.env` 填入有效的 DeepSeek Key |
| `AI service error: 422` | FastAPI 收到不合法数据 | 检查传入的 `healthData` 是否为空数组 |
| `AI service error: Connection refused` | FastAPI 未启动 | 启动 FastAPI（见上方启动步骤） |
| `AI response parse error` (502) | Agent 返回了非 JSON | 检查 FastAPI 终端日志，调优 `prompts/*.md` |
| 画像显示"无数据"或内容空泛 | 健康数据太少 | 至少录入5条不同类型的数据 |
| 登录后 AI 接口返回空结果 | `user_id` 与健康数据的 `user_id` 不匹配 | `SELECT DISTINCT user_id FROM health_data;` 确认 |
