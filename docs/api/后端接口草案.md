# 后端接口草案 v1

本文档是前后端联调用的第一版接口契约。接口命名和字段允许在评审后调整，但调整后必须同步更新本文档。

## 1. 通用约定

### 1.1 基础地址

本地开发：

```text
http://127.0.0.1:8080
```

### 1.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

约定：

- `code = 0` 表示成功。
- 非 0 表示失败。
- `message` 给前端展示或调试。
- `data` 返回具体业务数据。

### 1.3 分页响应格式

```json
{
  "items": [],
  "page": 1,
  "size": 10,
  "total": 0
}
```

约定：

- `page` 从 1 开始。
- `size` 默认 10。
- 时间字段统一使用 `yyyy-MM-dd HH:mm:ss`。

### 1.4 登录凭证

登录成功后返回 `token`。除注册、登录外，业务接口默认要求请求头携带：

```http
Authorization: Bearer <token>
```

## 2. 认证与用户接口

### 2.1 用户注册

```http
POST /api/auth/register
```

请求：

```json
{
  "phone": "13800138000",
  "username": "test_user",
  "password": "123456",
  "confirmPassword": "123456"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": 1
  }
}
```

规则：

- 手机号唯一。
- 密码和确认密码必须一致。
- 密码必须加密存储。
- 该接口只用于普通用户注册。
- 请求体不接受 `role` 字段，后端固定创建 `USER` 角色。
- 管理员账号第一阶段通过初始化 SQL 创建，不开放公开注册接口。

### 2.2 用户登录

```http
POST /api/auth/login
```

请求：

```json
{
  "phone": "13800138000",
  "password": "123456"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "user": {
      "userId": 1,
      "username": "test_user",
      "phone": "138****8000",
      "role": "USER"
    }
  }
}
```

### 2.3 查询当前用户信息

```http
GET /api/users/me
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": 1,
    "username": "test_user",
    "phone": "138****8000",
    "birthDate": "2000-01-01",
    "gender": "MALE",
    "height": 175.5,
    "role": "USER"
  }
}
```

### 2.4 修改当前用户信息

```http
PUT /api/users/me
```

请求：

```json
{
  "username": "new_name",
  "birthDate": "2000-01-01",
  "gender": "MALE",
  "height": 175.5
}
```

## 3. 健康数据接口

### 3.1 新增健康数据

```http
POST /api/health-data
```

请求：

```json
{
  "dataType": "WEIGHT",
  "dataValue": "70.5",
  "recordTime": "2026-05-24 08:30:00"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "dataId": 1
  }
}
```

第一版数据类型建议：

- `BLOOD_PRESSURE`
- `BLOOD_GLUCOSE`
- `WEIGHT`
- `EXERCISE_MINUTES`
- `SLEEP_HOURS`

### 3.2 查询健康数据

```http
GET /api/health-data?dataType=WEIGHT&startTime=2026-05-01 00:00:00&endTime=2026-05-24 23:59:59&page=1&size=10
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "dataId": 1,
        "dataType": "WEIGHT",
        "dataValue": "70.5",
        "recordTime": "2026-05-24 08:30:00"
      }
    ],
    "page": 1,
    "size": 10,
    "total": 1
  }
}
```

### 3.3 删除健康数据

```http
DELETE /api/health-data/{dataId}
```

规则：

- 普通用户只能删除自己的数据。
- 管理员是否允许删除用户健康数据，后续评审决定。

### 3.4 批量导入健康数据

```http
POST /api/health-data/import
Content-Type: multipart/form-data
```

表单字段：

- `file`: CSV 文件。

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "importedCount": 5
  }
}
```

说明：

- CSV 模板当前由前端静态文件 `/health_data_template.csv` 提供。
- CSV 表头固定为 `dataType,dataValue,unit,recordTime`。
- `recordTime` 格式固定为 `yyyy-MM-dd HH:mm:ss`。
- 任一数据行格式错误时整文件导入失败，不写入部分数据。

### 3.5 健康趋势数据

```http
GET /api/health-data/trends?dataType=WEIGHT&startTime=2026-05-01 00:00:00&endTime=2026-05-24 23:59:59
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "dataType": "WEIGHT",
    "points": [
      {
        "date": "2026-05-24",
        "value": "70.5"
      }
    ]
  }
}
```

## 5. AI 接口

后端返回 mock 数据用于调试。

### 5.1 生成健康画像

```http
POST /api/ai/profile
```

请求：

```json
{
  "timeRange": "LAST_7_DAYS"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "profileId": 1,
    "healthTags": ["睡眠不足"],
    "riskLevel": "LOW",
    "analysis": "近期睡眠时长偏少，建议保持规律作息。"
  }
}
```

### 5.2 生成健康计划

```http
POST /api/ai/plan
```

请求：

```json
{
  "healthNeed": "改善睡眠"
}
```

### 5.3 生成健康知识推送

```http
POST /api/ai/knowledge-push
```

请求：

```json
{
  "profileId": 1
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "pushIds": [1, 2]
  }
}
```

说明：

- 第一版可以根据健康标签从 `knowledge` 表匹配内容，并写入 `knowledge_push`。
- 后续对接 AI 服务后，可由 AI 服务返回推荐知识 ID 或推荐理由。

## 6. 健康知识接口

### 6.1 查询健康知识列表

```http
GET /api/knowledge?keyword=睡眠&tag=睡眠&page=1&size=10
```

响应字段建议：

- `knowledgeId`
- `title`
- `content`
- `relateTag`
- `source`
- `createTime`

### 6.2 查询我的知识推送

```http
GET /api/knowledge/pushes?page=1&size=10
```

响应字段建议：

- `pushId`
- `knowledgeId`
- `title`
- `relateTag`
- `pushTime`
- `isRead`

### 6.3 标记知识推送为已读

```http
PATCH /api/knowledge/pushes/{pushId}/read
```

规则：

- 普通用户只能修改自己的推送记录。

## 8. 管理员接口

管理员接口要求用户角色为 `ADMIN`。

第一阶段不提供公开的管理员注册接口。管理员账号由初始化 SQL 创建；后续如果需要新增管理员账号，应通过受保护的管理员接口实现。

### 8.1 用户列表

```http
GET /api/admin/users?keyword=138&page=1&size=10
```

### 8.2 启用或禁用用户

```http
PATCH /api/admin/users/{userId}/status
```

请求：

```json
{
  "status": "DISABLED"
}
```

### 8.3 平台统计

```http
GET /api/admin/statistics?startDate=2026-05-01&endDate=2026-05-24
```

响应字段建议：

- 用户总数。
- 健康数据总数。
- 风险等级分布。
- 每日数据录入量。

### 8.4 登录日志查询

```http
GET /api/admin/login-logs?userId=1&result=1&page=1&size=10
```

响应字段建议：

- `logId`
- `userId`
- `phone`
- `loginIp`
- `loginTime`
- `loginResult`

### 8.5 健康知识管理

查询：

```http
GET /api/admin/knowledge?keyword=睡眠&page=1&size=10
```

新增：

```http
POST /api/admin/knowledge
```

```json
{
  "title": "睡眠健康建议",
  "content": "保持规律作息，睡前减少电子屏幕使用。",
  "relateTag": "睡眠不足",
  "source": "系统内置"
}
```

修改：

```http
PUT /api/admin/knowledge/{knowledgeId}
```

删除：

```http
DELETE /api/admin/knowledge/{knowledgeId}
```

规则：

- 健康知识管理接口仅管理员可访问。
- 如果知识已经被推送，删除策略后续评审决定；第一版建议软删除或禁止删除。
