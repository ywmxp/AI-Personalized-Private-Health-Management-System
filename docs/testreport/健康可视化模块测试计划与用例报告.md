## 一、测试计划
### 1.1 测试目的
验证健康可视化模块的趋势图渲染、数据统计、报告导出功能是否符合需求规格说明书与设计文档要求，确保图表数据与原始健康数据一致、统计指标计算准确、导出文件格式与内容正确，为联调测试与项目验收提供质量依据。

### 1.2 测试范围
#### 1.2.1 覆盖功能点
- **Must级**：健康趋势图展示（数据类型筛选、时间范围筛选、折线图/饼图渲染、统计指标计算）
- **Should级**：健康报告导出（pdf格式）
#### 1.2.2 不覆盖范围
- 健康数据录入、删除、批量导入（归属健康数据管理模块）
- AI健康画像生成、健康计划生成（归属AI健康分析模块）

### 1.3 测试环境
| 环境项 | 配置说明 |
| ---- | ---- |
| 操作系统 | Windows 10 及以上 |
| 浏览器 | Edge 最新版 |
| 后端服务 | Spring Boot 3.5.7，服务端口 8080 |
| 数据库 | MySQL 8.4，执行最新建表与初始化脚本 |
| 前端环境 | Vue 3.5，关闭 Mock 模式（`VITE_MOCK_API=false`） |
| 代码分支 | `dev` 分支最新代码 |
| 测试账号 | 预置普通用户测试账号（USER角色），账户内包含多类历史健康数据 |

### 1.4 测试策略
1. **测试类型**：黑盒功能测试、数据准确性校验、图表渲染验证、兼容性测试
2. **设计方法**：等价类划分法、边界值分析法、异常场景法、数据比对法
3. **执行优先级**：
   - P0（最高）：趋势图筛选、图表渲染、统计指标计算（Must级核心功能）
   - P1（次高）：健康报告导出、权限校验、边界异常场景（Should级功能）
4. **执行方式**：前端页面操作 + 接口返回数据校验 + 数据库原始数据三方比对

### 1.5 缺陷管理规范
1. **缺陷分级**
   - 严重：图表无法渲染、数据统计错误、导出文件损坏、权限失效
   - 一般：筛选条件失效、图表布局错乱、导出内容缺失、自适应异常
   - 轻微：样式细节问题、文案不规范、提示不友好
2. **闭环流程**：缺陷登记 → 分配对应开发修复 → 修复后拉取最新代码复测 → 复测通过关闭缺陷 / 不通过退回重改

### 1.6 测试通过标准
1. Must级功能无严重、一般级别缺陷，轻微缺陷不超过2个；
2. Should级功能无严重级别缺陷，一般缺陷不超过2个；
3. 图表展示数据、统计指标与数据库原始健康数据完全一致，无计算错误；
4. 导出文件格式正确、内容完整、中文无乱码。
