## 一、测试计划
### 1.1 测试目的
验证健康数据管理模块的功能逻辑、数据交互是否符合需求规格说明书与设计文档要求，确保手动录入、查询筛选、批量导入、数据删除等核心功能运行正常，数据存储准确，为联调测试与项目验收提供质量依据。

### 1.2 测试范围
#### 1.2.1 覆盖功能点
- **Must级**：健康数据手动录入、健康数据查询（筛选+分页）、健康数据删除
- **Should级**：健康数据CSV批量导入
#### 1.2.2 不覆盖范围
- 健康趋势可视化图表（归属健康可视化模块）
- 健康数据关联AI分析（归属AI健康分析模块）

### 1.3 测试环境
| 环境项 | 配置说明 |
| ---- | ---- |
| 操作系统 | Windows 10 及以上 |
| 浏览器 | Edge 最新版 |
| 后端服务 | Spring Boot 3.5.7，服务端口 8080 |
| 数据库 | MySQL 8.4，执行最新建表与初始化脚本 |
| 前端环境 | Vue 3.5，关闭 Mock 模式（`VITE_MOCK_API=false`） |
| 代码分支 | `dev` 分支最新代码 |
| 测试账号 | 自创建账号 |

### 1.4 测试策略
1. **测试类型**：黑盒功能测试、接口一致性校验、数据库数据核对、权限安全测试
2. **设计方法**：等价类划分法、边界值分析法、异常场景法、越权测试法
3. **执行优先级**：
   - P0（最高）：手动录入、数据查询、数据删除（Must级核心功能）
   - P1（次高）：批量导入（Should级功能）
4. **执行方式**：前端页面操作 + 数据库逐字段核对

### 1.5 缺陷管理规范
1. **缺陷分级**
   - 严重：核心功能阻断、数据错误/丢失、系统崩溃
   - 一般：功能逻辑异常、交互不符合需求、分页/筛选失效
   - 轻微：样式错乱、文案不规范、提示不友好
2. **闭环流程**：缺陷登记 → 分配对应开发修复 → 修复后拉取最新代码复测 → 复测通过关闭缺陷 / 不通过退回重改

### 1.6 测试通过标准
1. Must级功能无严重、一般级别缺陷，轻微缺陷不超过2个；
2. Should级功能无严重级别缺陷，一般缺陷不超过2个；
3. 核心业务流程全链路通畅，前后端数据与数据库表数据完全一致；
