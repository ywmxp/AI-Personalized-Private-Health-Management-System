## 一、测试计划
### 1.1 测试目的
验证管理员管理模块的用户账号管理、平台健康数据统计功能是否符合需求规格说明书与设计文档要求，确保用户列表查询准确、账号状态管理生效、统计数据与底层数据一致、权限隔离合规，为联调测试与项目验收提供质量依据。

### 1.2 测试范围
#### 1.2.1 覆盖功能点
- **Must级**：用户管理（用户列表查询、多条件筛选、分页、账号启用/禁用）、健康数据统计（用户总量统计、数据录入量统计、健康风险分布统计、图表展示）
#### 1.2.2 不覆盖范围
- 普通用户健康数据录入、删除、分析（归属健康数据管理、AI健康分析模块）
- 用户注册、登录、个人信息修改（归属用户认证模块）
- 健康提醒、知识推送等用户侧业务功能

### 1.3 测试环境
| 环境项 | 配置说明 |
| ---- | ---- |
| 操作系统 | Windows 10 及以上 |
| 浏览器 | Edge 最新版 |
| 后端服务 | Spring Boot 3.5.7，服务端口 8080 |
| 数据库 | MySQL 8.4，执行最新建表与初始化脚本 |
| 前端环境 | Vue 3.5，关闭 Mock 模式（`VITE_MOCK_API=false`） |
| 代码分支 | `dev` 分支最新代码 |
| 测试账号 | 预置管理员测试账号（ADMIN角色）、普通用户测试账号（USER角色，包含启用/禁用两种状态） |

### 1.4 测试策略
1. **测试类型**：黑盒功能测试、数据准确性校验、权限安全测试
2. **设计方法**：等价类划分法、边界值分析法、异常场景法、数据比对法
3. **执行优先级**：
   - P0（最高）：用户列表查询、账号状态管理、核心统计数据展示（Must级核心功能）
   - P1（次高）：多条件筛选、分页、权限校验、边界异常场景
4. **执行方式**：前端页面操作 + 接口返回数据校验 + 数据库原始数据三方比对

### 1.5 缺陷管理规范
1. **缺陷分级**
   - 严重：权限越权漏洞、统计数据错误、账号状态管理失效、系统崩溃
   - 一般：筛选条件失效、分页异常、图表渲染错误、操作无反馈
   - 轻微：样式细节问题、文案不规范、提示不友好
2. **闭环流程**：缺陷登记 → 分配对应开发修复 → 修复后拉取最新代码复测 → 复测通过关闭缺陷 / 不通过退回重改

### 1.6 测试通过标准
1. Must级功能无严重、一般级别缺陷，轻微缺陷不超过2个；
2. 用户列表、统计数据与数据库原始数据完全一致，无统计、计算错误；
3. 管理员权限隔离生效，普通用户无法访问管理员页面与接口，无越权漏洞；
4. 图表渲染正常，页面交互符合设计要求，无控制台报错。
